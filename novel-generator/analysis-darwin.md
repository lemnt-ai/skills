### D8 — 实测表现（权重 25）[dry_run]

使用 3 个测试 prompt 进行干跑推演：

#### TP1: 完整创作（都市悬疑，心理医生，10万字）

**预期执行：**
- Step 0 → 判断为"完整创作"模式，输出执行计划
- Step 1 → 输出世界观（现代都市 + 超能力体系）、人物表（主角心理医生 + 案件相关角色）、主线（三幕式悬疑结构）
- Step 2 → 按中篇 3000-5000 字/章，规划 20-30 章
- Step 3 → 逐章写作 + 校验

**价值判断：** Skill 提供了明确的流程约束，比无 skill 的基线（直接写剧情）更能保证结构完整。字数阈值和校验模板在大纲阶段就能帮助控制节奏。

**评分：8/10** — 流程完整，但缺少一个"快速启动"路径，用户可能不想确认 5 次才看到正文。

#### TP2: 续写第4章（已有前3章）

**预期执行：**
- Step 0 → 判断为"续写"模式
- Step 2 → 只输出第 4 章的分章大纲
- Step 3 → 写作 + 校验

**价值判断：** 对比基线（直接写第 4 章），skill 强制的"读取前 3 章摘要"和"校验与设定一致性"有效防止续写时出现设定矛盾。

**评分：9/10** — 续写流程清晰，前文摘要和人物状态跟踪是亮点。唯一缺失：未明确说明"用户提供的前三章如何输入"。

#### TP3: 重写第5章打斗场面

**预期执行：**
- Step 0 → 判断为"修改"模式
- Step 3 → 定位第 5 章，确认修改要求，重写
- 校验 → 执行单章校验
- 标注影响范围

**价值判断：** "标注修改影响范围"是关键设计——对比基线（直接改完一章了事），这个步骤能防止修改蝴蝶效应。

**评分：8/10** — 修改模式设计完整。缺点：重写后"检查与下一章衔接"没有在流程中明确体现。

#### 综合判定

| 指标 | 判定 |
|------|:----:|
| 完成用户意图 | ✅ 三个场景均能覆盖 |
| 相比 baseline 提升 | ✅ 章节校验、伏笔追踪、全局一致性检查是独有价值 |
| skill 引入的负面影响 | ⚠️ 流程较长，用户可能需要多次确认，有摩擦感 |
| 最大亮点 | 校验体系（字数+内容+剧情）+ 伏笔回收追踪 |

**得分：8/10** → 贡献值：20.0
